Front End

Front-end site for Stadion Digital Sports platform

Summary

This is a standalone site running on a Node Express server. It's acts a static site to allow fast prototyping and development of modules without being slowed down by real data from a CMS. All assets to be used in the related Stadion platform repository are generated and minified io an output folder dist which can then be copied over to the platform. This includes handlebars templates which are transformed in the platform using Nustache

Backend:

Using Expressjs, the site is rendered from a rest api, Apiary.io. The data supplied from this rest endpoint matches the data structure that is used in the actual Stadion platform. Each module eg: Content Listing module, has it's own data set and is documented. See the api here

Frontend:

The site is completely modularised and each component has it's own assets and handlebars template. This allows us to easily add and remove components as needed.

Css is built using stylus and follows BEM syntax and OOCSS patterns similar to those outlined here as well as loosly following Google's html and css style guide. More details can be found in the styles foldericons folder

Javascript is built using Browserify. This allows to to write concise, modularised, Nodejs style javascript following commonjs practices. A vendor.js file is built out seperately and any third party libraries that have issues running inside Browserify can be built into this file and added to the Gulp commmand.

Icons are generated inline in css files from svg files using a mix of Filament Group's Grunticon tool and it's Gulp version More details can be found in the icons foldericons folder.

All images, icons, and built css and javascript files are output to the dist folder with the ''Gulp prod'' command.
